Class WsWithLongPollAuthenticationHandler

public final class WsWithLongPollAuthenticationHandler extends Object implements HttpAuthenticationHandler, WsAuthenticationHandler, PanelUserAuthenticationHandler
A combined authentication handler for a WS with long polling webserver handler

Accepts authentication via the following methods:

  • Header Authorization: Basic base64[User:Pass]
  • Cookie panellogin=base64[User:Pass]
  • WS or HTTP long poll frame {"authenticate": "Authentication Token"}
NOTE: Password should be a hex-encoded SHA-256 hash. See Digest.sha256(String)

NOTE: If both the Authorization: Basic header and panellogin cookie are provided, the Authorization: Basic header takes priority

NOTE: For HTTP long polling, authenticate frames must be wrapped in a JSON array, and response frames will be wrapped in a JSON array

When authenticating via Web Socket or HTTP long poll authenticate frame, the following response frame is sent:
  • On Failure: {"authresult": "false", "authtype": "none", "sessionId": null}
  • On Success: {"authresult": "true", "authtype": "read/write", "sessionId": "base64 session ID"}
If an authenticate frame sent via Web Socket fails or throws an exception, the connection is closed

A session can be resumed when authorizing via Authorization header or cookie by also including the header SessionID: sessionId with a valid session ID as returned in a authresult success frame

HTTP Response Codes:
  • HttpResponseStatus.OK - authenticate frame passed validation. See response JSON for session information
  • HttpResponseStatus.UNAUTHORIZED - All possible authentication methods failed validation
  • HttpResponseStatus.INTERNAL_SERVER_ERROR - Failed to parse first JSON array entry as a JSON object, or a database error occurred. See core-error log or bot console for information
Web Socket Close Codes:
  • WebSocketCloseStatus.POLICY_VIOLATION - All possible authentication methods failed validation
  • WebSocketCloseStatus.INTERNAL_SERVER_ERROR - Failed to parse frame text as a JSON object, or a database error occurred. See core-error log or bot console for information
