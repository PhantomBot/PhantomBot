A combined frame handler for a WS with long polling webserver endpoint

Authentication is handled via a WsWithLongPollAuthenticationHandler. See documentation for that class on how to perform authentication and the authentication responses

When initiating a Web Socket connection or performing a HttpMethod.GET, the query params afterTimestamp=millis and afterSequence=sequence can be used to signal where the stream should resume and trigger message replay, if available. All messages after the specified timestamp/sequence combination will be replayed if the GC hasn't collected them, excluding the message which exactly matches

The values can be updated without replaying messages by adding the optional skipTimestamp and skipSequence parameters in the metadata of a frame. See the frame format below

NOTE: It is not possible to return to an earlier timestamp/sequence once the query param or metadata param is sent. The messages are permanently released to the GC

When frames are sent via Web Socket:
  1. If the frame is a RFC 6455 PING frame, netty automatically responds with a RFC 6455 PONG frame
  2. If the frame is a RFC 6455 PONG frame, the WSServerPinger attached to the channel refreshes the timeout
  3. If the frame is a TEXT frame containing a valid JSON object, it is passed onto handleMessage(ChannelHandlerContext, JSONObject)
  4. If the frame is a TEXT frame, but does not contain a valid JSON Object, the JSONException is logged to core-error and then dropped
  5. All other frames are dropped
When frames are sent via HTTP:
  1. If the HTTP method is not HttpMethod.GET or HttpMethod.POST, HttpResponseStatus.METHOD_NOT_ALLOWED is returned
  2. If the HTTP method is HttpMethod.POST and the body fails to parse as a JSON array, or the JSON array is empty, HttpResponseStatus.BAD_REQUEST is returned and any JSONException is logged to core-error
  3. If the HTTP method is HttpMethod.POST and an exception occurs while processing a frame, HttpResponseStatus.INTERNAL_SERVER_ERROR is returned and the exception is logged in core-error
  4. If the HTTP method is HttpMethod.POST and an element in the JSON array is not a JSON object or the JSON object does not conform to the specification below, HttpResponseStatus.UNPROCESSABLE_ENTITY is returned and any JSONException exception is logged in core-error
  5. If the HTTP method is HttpMethod.POST and all frames are processed successfully, HttpResponseStatus.ACCEPTED is returned
  6. If the HTTP method is HttpMethod.GET and outbound frames are already queued, the request completes immediately with the frames
  7. If the HTTP method is HttpMethod.GET and no outbound frames are available, the server holds the request open for up to ctxTimeout, then completes with an empty JSON array if there are still no outbound frames available
NOTE: If the HTTP method is HttpMethod.POST and a HttpResponseStatus.INTERNAL_SERVER_ERROR or HttpResponseStatus.UNPROCESSABLE_ENTITY response is returned, the response body will be a JSON array containing frames which errored. Any frames not in the array should be considered successfully transmitted. [{"index": arrayIndex, "code": responseCode, "status": "response code name", "frame": frame}, ...]

NOTE: If the HTTP method is HttpMethod.POST and a response of HttpResponseStatus.ACCEPTED or HttpResponseStatus.BAD_REQUEST is returned, or a HttpResponseStatus.METHOD_NOT_ALLOWED response is returned for any other HTTP method, an empty JSON array is returned in the body

Server -> Client frame format: {"metadata": {"timestamp": enqueueTimestampMillis, "sequence": sequenceWithinTimestamp}, "data": dataJsonObject}

For HttpMethod.GET, all available frames are returned in a JSON array. For Web Socket, each frame is sent individually as a TEXT frame

Client -> Server frame format: {"metadata": {"timestamp": enqueueTimestampMillis, "sequence": sequenceWithinTimestamp, "skipTimestamp?": timestampMillis, "skipSequence?": sequence}, "data": dataJsonObject}

For HttpMethod.POST, a JSON array must be sent, containing the described JSON object as values. Multiple frames may be send in a single JSON array. For Web Socket, each frame must be sent individually as a TEXT frame
